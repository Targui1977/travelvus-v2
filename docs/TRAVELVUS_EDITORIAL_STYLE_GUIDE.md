# TRAVELVUS EDITORIAL STYLE GUIDE

**Version:** V2.0 — July 2026
**Status:** Authoritative. All future pages must follow this guide.

---

## VOICE

Travelvus writes like a calm, evidence-first editorial publication — not a travel blog, not an SEO site, not a booking platform.

**We are:** Precise. Measured. Transparent. Helpful. Never salesy.

**We sound like:** Financial Times meets Apple documentation. Data-driven but human. Confident but never arrogant. We explain reality. The reader decides.

### The core rule

> Travelvus never tells people what to do. Travelvus explains what the evidence shows. The traveller makes the decision.

---

## TONE

| Situation | Tone |
|-----------|------|
| Verdict / Recommendation | Calm, evidence-backed, never triumphant |
| Warning / Limitation | Honest, clear, never alarmist |
| Example / Scenario | "Here is what the numbers show." — not "You should..." |
| FAQ | Direct. One clear answer. No filler. |
| Methodology | Transparent. "We used X data from Y source on Z date." |
| CTA | Invitation, not instruction. "Compare your journey" not "Book now." |

### Forbidden tones

- ❌ Marketing: "The best airport ever!"
- ❌ SEO filler: "In today's fast-paced world..."
- ❌ Hype: "You won't believe which airport wins!"
- ❌ False urgency: "Book before prices rise!"
- ❌ Generic: "Whether you're a business traveller or..."

## VOCABULARY

### Use these words

| Instead of | Use |
|-----------|-----|
| Best airport | Recommended airport (or "our recommendation") |
| Winner | Recommended option / better choice |
| Perfect | Clear advantage / strong evidence |
| Amazing | Significant / substantial |
| Always | In most scenarios / typically |
| Never | Rarely / in limited circumstances |
| Guaranteed | Supported by evidence / data shows |
| Book now | Compare your journey |
| Click here | See the comparison / View the evidence |
| Cheap | Lower-cost / budget |
| Expensive | Higher-cost / premium |

### Forbidden words (never use)

- ❌ Best / Worst (oversimplifies)
- ❌ Guaranteed / Promise (we don't guarantee)
- ❌ Amazing / Incredible / Unbelievable (marketing)
- ❌ Always / Never (exceptions exist)
- ❌ Cheap (dismissive; use "lower-cost" or "budget")
- ❌ Everyone / Nobody (hyperbole)
- ❌ Perfect / Flawless (nothing is perfect)
- ❌ Secret / Hack / Trick (clickbait)
- ❌ You must / You should (prescriptive)

## HEADLINE RULES

### Flagship Guides
Pattern: `[Option A] vs [Option B]: The Complete Decision Guide`

Example: "Heathrow vs Gatwick: The Complete Decision Guide"

### Neighbourhood Guides
Pattern: `Best Airport for [Place] — [Question]?`

Example: "Best Airport for Manhattan — JFK, Newark or LaGuardia?"

### Decision Pages
Pattern: `[Origin] to [Destination]: [Airport A] or [Airport B]?`

Example: "Berlin to London: Heathrow or Stansted?"

### Rules
- One question per headline
- Max ~70 characters
- No clickbait ("You won't believe...")
- No false certainty where uncertainty exists

## RECOMMENDATION LANGUAGE

### Strong evidence (very-high / high confidence)

> "The evidence points to Heathrow. The transfer is shorter and more reliable — and the total journey cost is lower once baggage is included. This recommendation is supported by TfL and National Rail data."

### Conditional recommendation (moderate / conditional)

> "For most travellers, Heathrow is the better choice. But the right airport depends on where you are staying and which airline you fly. Our neighbourhood guide below explains when each airport wins."

### Too close to call

> "The airports are separated by only 5 minutes of total journey time in our comparison. This is effectively a tie — your airline preference, flight schedule, and destination within the city should determine your choice."

## CTA RULES

### Standard CTAs

| Context | CTA |
|---------|-----|
| Hero (guide) | Compare your journey → |
| Hero (decision page) | Compare your flight → |
| Hub | Reveal the recommendation → |
| After verdict | See the full comparison → |
| Related section | Continue reading → |

### Rules
- One CTA per section
- Never "Book now" or "Buy tickets"
- Never more than 4 words
- Always an invitation, never an instruction

## WARNING LANGUAGE

### Standard warning types

| Type | Label | Example |
|------|-------|---------|
| Important note | Travelvus note | "This recommendation assumes weekday travel." |
| Assumption | Editorial assumption | "Ticket prices are illustrative." |
| Limitation | When this changes | "The recommendation changes if you check baggage." |
| Scenario context | Typical scenario | "Based on a solo traveller with cabin bag only." |

### Rules
- One warning type per context
- Never invent new warning labels page-by-page
- Always explain WHY something is noted

## DECISION LANGUAGE

### Every recommendation must explain:

1. **What** the evidence shows
2. **Why** one option is recommended
3. **When** the recommendation changes
4. **How** certain we are

### Example (good):

> "Heathrow is our recommendation for this route. The transfer is 35 minutes by Elizabeth Line — faster and cheaper than Stansted's 50-minute coach. This recommendation assumes off-peak travel with one checked bag. It changes if you are travelling with hand luggage only."

### Example (bad):

> "Heathrow wins. Book now."

---

## FAQ RULES

- One clear answer per question. No preamble.
- Maximum 3 sentences per answer.
- State the evidence first, then the conclusion.
- Never: "It depends..." without explaining what it depends on.

### Example (good):

> **Q: Which London airport is cheapest?**
>
> Stansted and Luton typically have the lowest ticket prices because they serve budget airlines. However, the total journey — including baggage and airport transfer — is often cheaper through Heathrow. Compare your specific flights to see the real cost.

---

## CURRENCY RULES

- Local destination costs use local currency (GBP for London, USD for NY, EUR for Paris)
- Approximate values use ~ prefix
- Never mix currencies without explanation
- Money values use the Money component, never raw string concatenation
